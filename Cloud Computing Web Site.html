مصطلحات مهمة

IaaS
البنية التحتية كخدمة — آلات افتراضية، شبكات.
PaaS
المنصة كخدمة — بيئة لتشغيل التطبيقات دون إدارة البنية.
SaaS
البرمجيات كخدمة — برامج جاهزة عبر الإنترنت.
VPC
شبكة افتراضية خاصة—عزل الشبكة داخل السحابة.

أسئلة شائعة

ما الفرق بين IaaS وPaaS؟ +
IaaS يوفر الخوادم والشبكات والبنية التحتية. PaaS يوفر بيئة تشغيل ويدير الطبقات الأدنى من البنية لك.
هل السحابة آمنة؟ +
نعم عند اتباع ممارسات الأمن: التشفير، التحكم بالوصول، مراقبة النشاط، وتحديث السياسات بشكل دوري.

20 أمر من أوامر Git

  1. git init — إنشاء مستودع جديد
  2. git clone — نسخ مستودع موجود
  3. git add — إضافة الملفات للـ staging area
  4. git commit — تسجيل تغييرات الملفات
  5. git status — عرض حالة المستودع
  6. git log — عرض سجل التعديلات
  7. git branch — إدارة الفروع
  8. git checkout — الانتقال بين الفروع
  9. git merge — دمج الفروع
  10. git pull — جلب وتحديث المستودع
  11. git push — رفع التعديلات
  12. git remote — إدارة المستودعات البعيدة
  13. git fetch — جلب التغييرات بدون دمج
  14. git reset — التراجع عن التعديلات
  15. git revert — إنشاء commit لعكس التغييرات
  16. git tag — إضافة علامة على commit
  17. git stash — حفظ تغييرات مؤقتة
  18. git diff — عرض الفروقات بين التعديلات
  19. git rm — إزالة ملفات من المستودع
  20. git config — إعدادات Git

إضافة مشاركين (Collaborators)

لإضافة مشاركين إلى مستودعك على GitHub، اتبع الخطوات التالية:

  1. افتح مستودعك على GitHub.
  2. اضغط على Settings أعلى الصفحة.
  3. من القائمة الجانبية اختر Collaborators & teams.
  4. في خانة البحث، اكتب اسم المستخدم أو البريد الإلكتروني للشخص الذي تريد إضافته.
  5. اضغط على Add collaborator بجانب اسم المستخدم.
  6. سينتظر GitHub قبول الدعوة من الشخص المضاف.
  7. بمجرد قبول الدعوة، سيتمكن المشارك من الوصول إلى المستودع وفقًا للأذونات المحددة.
